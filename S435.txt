## üß© How it works (technical summary)

- The code uses simple regular expressions in `kgeb/extractors.py` to extract:
  - Person entities from lines like `John Doe, age 32, works at OpenAI as a Researcher.`
  - Company entities from employer occurrences and industry lines
  - Project entities from `Project Alpha started on ... ends on ...` patterns
  - Relations such as `WorksAt`, `Manages` and derived `OwnsProject` (based on person's company)
- Deduplication: unique items are merged by name, and missing attributes merged when possible
- The evaluator:
  - `evaluate_entities()` computes precision/recall/f1 if `entities_gold.json` present
  - `evaluate_relations()` computes precision/recall/f1 if `relations_gold.json` present
  - If no gold resources, returns:
    - Schema compliance (entities and relations)
    - Logical consistency (relations referencing valid entities)
  - Conflict detection: identifies contradictions for same entity name (e.g., different ages)
- Output:
  - `entities_output.json` (extracted entities)
  - `relations_output.json` (extracted relations)
  - `evaluation_report.json` (evaluation summary, schema compliance, conflicts and metrics)
